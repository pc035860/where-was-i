# WWI (Where Was I?)

> [繁體中文版](./README.zh-TW.md)

A CLI tool that scans AI coding agent sessions and displays their status with LLM-powered intent synthesis. When you're juggling multiple AI agents across different projects, WWI gives you a quick overview of what each one is doing.

## What It Does

WWI scans session files from three AI coding agents:

- **Claude Code** — reads from `~/.claude/projects/`
- **Codex** — reads from `~/.codex/sessions/`
- **Gemini CLI** — reads from `~/.gemini/tmp/`

For each active session, it shows:
- Which agent is running and which project it belongs to
- How recently the session was active (🟢 active < 20 min, 🟡 recent < 2 hr, ⚪ stale > 2 hr)
- A one-line intent summary generated by an LLM (what the agent is working on)

## Prerequisites

- [Bun](https://bun.sh/) >= 1.0.0
- An API key for at least one LLM provider:
  - **Gemini** (default): set `GEMINI_API_KEY` or `GOOGLE_API_KEY`
  - **OpenAI**: set `OPENAI_API_KEY`

## Installation

```bash
git clone <repo-url>
cd where-was-i
bun install
```

To register `wwi` as a global command:

```bash
bun link
```

After linking, you can run `wwi` directly from anywhere:

```bash
wwi status
wwi watch
```

## Usage

### One-shot status

```bash
bun run src/main.ts status
```

This scans all agent sessions, synthesizes intents via LLM, and prints a snapshot to the terminal.

### Watch mode (main use case)

```bash
bun run src/main.ts watch
```

A persistent TUI that refreshes every 2 seconds. Keyboard shortcuts:

| Key | Action |
|---|---|
| `q` / `Ctrl-C` | Quit |
| `s` | Toggle stale sessions |
| `a` | Toggle expand all (override adaptive height) |

### CLI Options

| Option | Description | Default |
|---|---|---|
| `--show-stale` | Include stale sessions (> 2 hr old) | `false` |
| `--no-intent` | Skip LLM intent synthesis | intent enabled |
| `--debug` | Show debug output (API timing, adapter info) | `false` |
| `-p, --provider <name>` | LLM provider: `gemini` or `openai` | `gemini` |
| `-m, --model <name>` | Override the model name | Gemini: `gemini-3.1-flash-lite-preview`, OpenAI: `gpt-4.1-mini` |

### Examples

```bash
# Use OpenAI instead of Gemini
bun run src/main.ts status -p openai

# Watch mode with a specific model
bun run src/main.ts watch -m gemini-2.5-flash

# Quick status without LLM calls
bun run src/main.ts status --no-intent

# Show everything including stale sessions
bun run src/main.ts status --show-stale
```

## How It Works

1. **Scan** — The scanner globs session files from each agent's known directory, filters out sessions older than 24 hours, and extracts project names from the session data.

2. **Extract Context** — For each active session, the context extractor reads the tail of the session file (JSONL for Claude/Codex, full JSON for Gemini) and pulls out recent user messages, assistant messages, and tool calls.

3. **Synthesize Intent** — The extracted context is fed into an LLM prompt that asks for a one-line summary of what the agent is currently working on. Results are cached to disk (`/tmp/wwi-intent-cache.json`) and debounced to avoid excessive API calls.

4. **Render** — The TUI draws a bordered box with color-coded agent cards showing project name, session ID, activity status, and intent summary.
